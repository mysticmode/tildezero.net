Detailed walkthrough on installing minimalist Arch Linux with i3

This article will walk you through the installation of Arch Linux in detail and how to configure the minimalist i3 setup.

When you boot into the ISO device, you will see a screen where it says: Install Arch Linux. Select that and you will see the boot sequence that enters into root@archiso ~ #. The ending # is one of the common symbols on Bash shell or sh, where:

  • # means you are operating the system as a root user.
  • $ means you are operating the system as a normal user.

Here in the initial setup of Arch Linux, the root user is the one who's going to setup the operating system, thus # is used. For brevity purpose, I will be writing the commands starting just with #.

WiFi configuration

Check for the network interface:

# ip link

You will get the list of network interfaces, for example with WiFi - it would be in the lines of wlo1. Now create and open a new file at /etc/wpa_supplicant/wpa_supplicant.conf with your editor of choice. Then type in your WiFi address and password as ssid and psk respectively.

  • WiFi address is ssid
  • WiFi password is psk

So, the wpa_supplicant.conf file will be in the format of: 

network: {
  ssid: <your-wifi-address>
  psk: <your-wifi-password>
}

Replace the <your-wifi-address> with your WiFi address. For example, my WiFi address is called "mystic" and password as "abcd", it will be: 

network: {
  ssid: "mystic",
  psk: "abcd"
}

When you are done with this, save and close the file. Then you have to say the wpa_supplicant command utility to point where your WiFi configuration is and run it as daemon or in other words, as a background process. In order to do that, you will call: 

wpa_supplicant -B -i wlo1 -c /etc/wpa_supplicant/wpa_supplicant.conf

where -B means background process, -i is your WiFi interface and finally -c is to point out the configuration file.

Now we need to use the DHCP client provided by Arch Linux to connect to the WiFi and by default - there will be dhclient. So, we will do: 

dhclient wlo1

This might take few seconds to connect to your WiFi, if dhclient is done, check your internet connection with: 

ping archlinux.org

The working connection will give the output similar to:

64 bytes from 93.184.216.34: icmp_seq=0 ttl=56 time=11.632 ms
64 bytes from 93.184.216.34: icmp_seq=1 ttl=56 time=11.726 ms
64 bytes from 93.184.216.34: icmp_seq=2 ttl=56 time=10.683 ms

Now do Ctrl+d in order to exit the WiFi connection check. By now, if all goes well for you - the WiFi connection has been setup in order to install Arch Linux!

Update the system clock

When our system enviornment connects to the internet, it will automatically detect the system clock via the systemd-timesyncd service which is enabled by default.

But make sure, the system clock is appropriate by doing:

timedatectl

This will print the system clock detail based on your timezone in the lines of: 

Local time: Thu 2017-09-21 16:08:56 CEST
Universal time: Thu 2017-09-21 14:08:56 UTC
RTC time: Thu 2017-09-21 14:08:56
Time zone: Europe/Warsaw (CEST, +0200)
System clock synchronized: yes
NTP service: active
RTC in local TZ: no
